# Vega Providers API Details

This project provides a collection of scraping modules (providers) for the Vega App. It extracts metadata and stream links from various movie and anime sources.

## How it Works
1. **Providers**: Individual scraping logic for different sites (e.g., VegaMovies, FlixHQ, HiAnime).
2. **Bundling**: TypeScript source code is bundled into standalone JavaScript files in the `dist/` folder.
3. **Server**: A development server serves these bundled files and a `manifest.json` registry.

## API Endpoints (Dev Server)
- `GET /manifest.json`: List of all providers.
- `GET /dist/{provider}/{file}`: The provider script itself.
- `GET /status`: Server and build status.
- `GET /providers`: List available provider names.
- `POST /build`: Manually trigger a rebuild.

## Key Features
- **Centralized Types**: Defined in `providers/types.ts`.
- **Auto-Rebuild**: Changes to provider source code are automatically detected and rebuilt.
- **Universal Interface**: Every provider implements a standard set of functions (`GetHomePosts`, `GetSearchPosts`, `GetMetaData`, `GetStream`).
- **Comprehensive Coverage**: Supports both movies and TV series (including episode extraction).
